# Documentation Rules

- Start README documents with the production problem being solved, not a project showcase framing.
- Do not use portfolio/demo language or first-person build narratives.
- Do not use emojis.
- Use Mermaid for architecture diagrams with supporting prose.
- Document realistic operational failure modes and recovery actions.
- Include explicit limitations with honest trade-offs.
- Keep instructions actionable for operators running the system in development and production-like environments.
